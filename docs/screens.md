# 画面仕様 — キョカミル

- 更新日: 2026-07-15(v2: トップ KV カルーセル・プレイスタイル絞り込み・メディアギャラリーを追加)
- 全ページ共通: モバイルファースト(配信準備中にスマホで確認するユースケースが主)。日本語 UI。

## デザイン方針: Steam ストアの模倣

ターゲットユーザー(配信者)が見慣れている **Steam ストアのレイアウト・要素配置を意図的に模倣する**。ユーザーは学習コストゼロで情報の場所を予測できることを狙う。

### トーン&カラー

- **ダークテーマ固定**(ライトテーマは作らない)。Steam 準拠の配色感:
  - 背景: 濃紺〜ダークグレー系(例: `#1b2838` 相当をベースに Tailwind でパレット定義)
  - パネル/カード: 背景よりやや明るい紺(例: `#2a475e` 相当)
  - 本文テキスト: オフホワイト(`#c7d5e0` 相当)、見出しは白
  - アクセント: Steam 的な明るいブルー(リンク・フォーカス)。可否バッジの緑/黄/赤/グレーは視認性優先でアクセントとは独立に定義
- グラデーションや装飾は控えめに。情報密度が高く実用的な「ストアページ」の質感

### レイアウトパターン(ページ別)

- **トップ**: Steam ストアトップの構成を踏襲 — ファーストビューは**大型 KV カルーセル(「注目&おすすめ」相当)**でゲームビジュアルを主役にする。テキスト主体のヒーローや検索ボックスの重複配置はしない(検索はヘッダーに一本化)。その下に「新着」「最近確認」をカプセル画像主体の横長セクションで並べる
- **ゲーム一覧**: Steam の検索結果ページを踏襲 — **横長の行リスト**(左: カプセル画像、中: タイトル・パブリッシャー・発売日、右: 配信可否/収益化可否バッジ+最終確認日)。**フィルタは PC では右サイドバー**(Steam 検索と同配置)、モバイルでは上部の折りたたみ
- **ゲーム詳細**: Steam のアプリページを踏襲 — 上部にパンくず(パブリッシャー > ゲーム名)、**ヘッダー画像+右カラムの情報ボックス**(Steam の購入エリアに相当する位置に「可否サマリー」の 2 大バッジを置く)、その下に詳細テーブル・備考・根拠が続く。モバイルでは縦積み
- **パブリッシャー詳細**: Steam のパブリッシャーページ風 — 上部に名前+包括情報、下にゲームの行リスト

### 模倣してよいもの / 禁止(重要)

| 模倣してよい | 禁止 |
|-------------|------|
| レイアウト・要素の配置・情報の階層 | Steam / Valve のロゴ・アイコン・商標画像の使用 |
| ダーク配色の方向性(色は独自定義) | Steam の CSS・画像アセット・フォントファイルの流用 |
| 行リスト・カプセル画像中心の見せ方 | 「Steam 公式/提携」と誤認されるヘッダーデザインや文言 |

サイト名「キョカミル」のロゴ(テキスト)をヘッダー左上に常時表示し、フッターの非提携表示(§0 Footer)と合わせて Steam 本体との混同を防ぐ。requirements.md §10・AGENTS.md ガードレール参照。

## 0. 共通コンポーネント

| コンポーネント | 内容 |
|---------------|------|
| `Header` | ロゴ(テキスト「キョカミル」)、検索ボックス(ゲーム名入力 → `/games?q=`)、ナビ(ゲーム一覧 / パブリッシャー / このサイトについて) |
| `Footer` | 静的ページリンク、Valve 帰属表示(「ゲーム情報・画像は Steam より取得しています。キョカミルは Valve Corporation とは無関係です」+ Steam へのリンク)、コピーライト |
| `StatusBadge` | enum 値を受け取り、日本語ラベル+色(緑/黄/赤/グレー、[database.md](database.md) §1 の対応表)で表示。色だけに依存せず必ずテキストを併記(アクセシビリティ) |
| `DisclaimerNote` | 免責注意書き。「本サービスは参考情報の提供であり、法的助言ではありません。配信前に必ず公式情報をご確認ください。」+該当ゲームの根拠リンク。**全ゲーム詳細ページに必須** |
| `GameCapsule` | トップページ用: カプセル画像(Steam CDN、`loading="lazy"`、alt=タイトル)+タイトル+可否バッジ 2 つのコンパクト表示 |
| `GameListRow` | 一覧用の横長行(Steam 検索結果風): 左=カプセル画像、中=タイトル・パブリッシャー名・発売日・プレイスタイルアイコン、右=配信可否/収益化可否バッジ+最終確認日 |
| `HeroCarousel` | トップの大型 KV(Steam「注目&おすすめ」相当): header_image を大きく表示し、上にタイトル+配信可否/収益化可否バッジをオーバーレイ。左右矢印+下部ドットで切替、5 秒間隔の自動送り(ホバー/フォーカスで停止、`prefers-reduced-motion` では自動送りしない)。クリックで詳細ページへ |
| `MediaGallery` | 詳細ページ用(Steam アプリページのメディア欄相当): 大きなメインビュー+下にサムネイル行。トレーラー動画(あれば先頭、`<video controls preload="none" poster={thumbnail}>`)→スクリーンショットの順。すべて Steam CDN ホットリンク(steam-integration.md §3) |
| `AdSlot` | 広告プレースホルダー(architecture.md §7) |

## 1. トップ `/`

**検索ボックスはヘッダーの 1 箇所のみ**(ヒーロー内には置かない。重複配置は禁止)。

上から順に:

1. **HeroCarousel(ファーストビューの主役)**: 公開ゲームのうち追加日の新しい順に最大 5 件。header_image を大きく表示し、タイトル+可否バッジ 2 つをオーバーレイ。テキストより先にゲームビジュアルが目に入ることを最優先する
   - キャッチコピー(「このゲーム、収益化配信して大丈夫?」)はカルーセル上部の小さなセクション見出しに縮小して残す(Steam の「注目&おすすめ」ラベル相当の扱い)
   - 掲載 5 件未満の場合はある分だけ表示、1 件ならカルーセルなしの静的 KV
   - 将来: `featured` フラグによる手動選定に置き換え可能な実装にする(MVP は新着順で自動)
2. 新着ゲーム(追加日順 8 件、GameCapsule の横並びセクション)
3. 最近確認したゲーム(last_verified_at 降順 8 件、同上)
4. 「情報の見方」簡易説明(バッジの意味)+ジャンル・プレイスタイルの人気フィルタへのショートカットリンク(例: 「ソロで遊べるゲーム」→ `/games?mode=singleplayer`)

## 2. ゲーム一覧・検索 `/games`

- URL クエリパラメータ駆動(すべて任意、組み合わせ可):
  - `q`: キーワード(title / title_en / パブリッシャー name を ilike 部分一致)
  - `streaming`, `monetization`: approval_status 値でフィルタ
  - `publisher`: パブリッシャー slug
  - `genre`: ジャンル文字列
  - `mode`: プレイスタイル(database.md §1 の play_mode 値。カンマ区切りで複数指定可、OR 条件)
  - `sort`: `name`(default) / `verified`(last_verified_at 降順) / `added`(created_at 降順)
  - `page`: ページネーション(24 件/ページ)
- 表示は `GameListRow` の縦リスト(Steam 検索結果と同じ構造)
- フィルタ UI: PC では右サイドバー(Steam 検索と同配置)、モバイルでは上部の折りたたみ。サイドバーの構成(上から): 配信可否 / 収益化可否 / **プレイスタイル**(ソロ・オンライン対戦・オンライン協力・ローカルマルチ・MMO のチェックボックス) / **ジャンル**(登録済みジャンルから動的生成) / パブリッシャー
- 配信者の主ユースケース「自分の配信スタイルに合うゲームを探す」を想定し、プレイスタイルとジャンルは選択中の条件をチップ表示していつでも外せるようにする
- 結果 0 件時: 「見つかりませんでした」+お問い合わせページへの誘導(「掲載リクエストはこちら」)
- `published = true` のみ表示

## 3. ゲーム詳細 `/games/[slug]`(最重要ページ)

レイアウトは Steam アプリページを踏襲(デザイン方針参照)。PC は 2 カラム(左: メイン、右: 情報ボックス)、モバイルは縦積み。上から順に:

1. **パンくず**: パブリッシャー名(リンク) > ゲーム名
2. **MediaGallery(左カラム、Steam アプリページのメディア欄相当)**: トレーラー動画(あれば先頭)+スクリーンショット(最大 6 枚)のギャラリー。「どんなゲームか」がテキストなしで伝わることを目的とする。メディアが未取得のゲームは従来どおり header_image のみ表示
3. **タイトル部(左カラム、ギャラリー直下)**: タイトル(日/英併記)、運営による一言紹介(summary、1〜2 文、あれば)、発売日、ジャンル、プレイスタイル(アイコン+ラベル)、「Steam で見る」テキストリンク(`https://store.steampowered.com/app/{steam_app_id}`、steam_app_id がある場合のみ)
4. **可否サマリー(右カラムの情報ボックス、Steam の購入エリア相当の位置)**: 配信可否・収益化可否の 2 つを大きなバッジで表示。ファーストビューに収める(モバイルではギャラリー直後に配置し、スクロールなしで見えること)
5. **詳細テーブル(左カラム)**: ネタバレ制限 / 音楽・BGM / 切り抜き・アーカイブ / 事前申請 の 4 項目をバッジ付きで表示
6. **備考**: notes をそのまま表示(改行反映)。空なら非表示
7. **根拠**: sources を種別ラベル付きリストで表示(外部リンク、`rel="noopener noreferrer"`、`target="_blank"`)。noted_at があれば併記
8. **鮮度情報**: 「運営最終確認日: YYYY-MM-DD」を可否サマリーの情報ボックス内にも表示
9. **DisclaimerNote**(必須)
10. `AdSlot`

- パブリッシャー既定値の継承表示(database.md §4-2): 継承値には「パブリッシャー包括ガイドラインに準拠」の注記を付ける
- 存在しない slug / `published = false` は 404
- `generateMetadata`: title = `{title}は配信・収益化していい?ガイドラインまとめ | キョカミル`、description には配信可否・収益化可否の結論を含める
- JSON-LD(`VideoGame`)を埋め込む

## 7. 管理画面 `/admin/*`(認証必須)

デザインは簡素でよい(運営専用)。全ページ `noindex`。

### `/admin/login`
- メール+パスワードのログインフォームのみ

### `/admin`(ダッシュボード)
- 統計: 公開ゲーム数 / 下書き数 / パブリッシャー数
- **要再確認リスト**: `last_verified_at` が 60 日以上前の公開ゲーム一覧(古い順)。運用の月次再確認(requirements.md §4.2)を支える

### `/admin/games`
- 全ゲーム一覧(下書き含む)。公開状態・最終確認日で絞り込み。編集リンク

### `/admin/games/new`
- **Steam 取込フロー**: Steam App ID または ストア URL を入力 → 「Steam から取得」ボタン → Server Action が Storefront API を叩き、タイトル・パブリッシャー名・発売日・ジャンル・ヘッダー画像 URL・**プレイスタイル・スクリーンショット・トレーラー動画 URL**をフォームにプリフィル(steam-integration.md 参照)
- スクリーンショットはサムネイル付きチェックボックスで取捨選択(既定: 先頭 6 枚を選択済み)。プレイスタイルは自動判定結果を手動で修正可能
- 運営の一言紹介(summary、任意、200 文字まで)。**Steam の説明文をコピペしない**(プレースホルダーに注意書きを表示)
- パブリッシャーは既存から選択 or その場で新規作成
- 許諾情報 6 項目(select)+ notes(textarea)+ sources(動的な複数行入力: url / source_type / label / noted_at)
- slug は title_en から自動生成(編集可)
- 保存時バリデーション: published にするには sources 1 件以上+ last_verified_at 必須(database.md §4-1)

### `/admin/games/[id]`
- 編集フォーム(new と同一フォームコンポーネント)+「最終確認日を今日にする」ワンクリックボタン
- 保存時に該当公開ページを revalidate

### `/admin/publishers`, `/admin/publishers/new`, `/admin/publishers/[id]`
- パブリッシャーの CRUD(項目は database.md のカラム通り)
- 削除は掲載ゲームが 0 件の場合のみ可
