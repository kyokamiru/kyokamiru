# 要件定義書 — キョカミル

配信者がゲームの配信可否(ガイドライン・許諾情報)を一覧・検索できる Web サービス

- 作成日: 2026-07-14
- 更新日: 2026-07-14
- ステータス: ドラフト v0.2

---

## 1. 背景・課題

ゲーム実況・配信を行う際、各ゲームメーカーが公開する「配信ガイドライン」に従う必要があるが、以下の課題がある。

- ガイドラインはメーカーごとに散在しており、探すのに手間がかかる
- **特に海外ゲーム(Steam 配信タイトル)は、日本語で規約情報がまとまっているサイトがほぼ存在しない**。国内大手メーカーのガイドラインは既存のまとめが多いが、海外タイトルは EULA や公式 FAQ を英語で読み解く必要がある
- 「配信可否」だけでなく「収益化の可否」「ネタバレ制限」「音楽の扱い」など条件が複雑
- ガイドラインは予告なく改定されることがあり、古い情報を信じて配信すると規約違反のリスクがある
- 収益化済み配信者にとっては、収益化 NG のゲームを誤って配信すると BAN・収益剥奪・法的リスクに直結する

## 2. サービスコンセプト

**「このゲーム、収益化配信して大丈夫?」に 10 秒で答えるデータベース。**

ゲーム名で検索すると、配信可否・収益化可否・条件・根拠となる公式情報へのリンクが一目でわかる。

### 差別化ポイント

**Steam で配信されている海外タイトルの配信許諾情報を日本語で整理する、事実上唯一のデータベース**を目指す。国内大手メーカーの包括ガイドライン(任天堂・カプコン等)は既存情報が豊富なため、本サービスの主戦場としない(補完的に掲載はする)。

## 3. ターゲットユーザー

| 優先度 | ユーザー | ニーズ |
|--------|---------|--------|
| 主 | 収益化済みの個人配信者(YouTube / Twitch 等でメンバーシップ・投げ銭・広告収益あり) | 海外ゲームを収益化配信して問題ないか、条件付きか、を確実に知りたい |
| 副 | これから収益化を目指す配信者 | 事前にリスクを把握したい |

## 4. コンテンツ方針・登録優先順位

### 4.1 メインコンテンツ

**Steam で取り扱いのある海外ゲームのうち、配信規約情報が日本語でまとまっていないタイトル。**

登録優先順位:

1. Steam で人気(同時接続・ウィッシュリスト上位)かつ配信需要が高い海外タイトル
2. 配信可否がわかりにくい・誤解されやすいタイトル(EULA に配信条項があるもの、音楽権利が複雑なもの等)
3. 新作・話題作(発売直後は配信可否の検索需要が急増する)
4. 国内大手メーカーの包括ガイドライン(補完コンテンツ。網羅性のために掲載)

### 4.2 データ収集・運用方針

- **運営が公式情報(公式ガイドライン、EULA、公式 FAQ、公式 Discord/フォーラムでの開発者回答等)を確認し、手動で登録・更新する**(品質重視)
- 登録時は必ず一次情報の URL と確認日を記録する
- ゲームの基本メタデータ(タイトル、パブリッシャー、発売日、対応プラットフォーム)は **Steam Web API / Storefront API から取得**して登録の手間を削減。許諾情報のみ人力で調査・入力する
- 定期的(月次目安)に既存データの再確認を行い、「最終確認日」を更新する

## 5. MVP 機能要件

MVP は **検索・一覧・詳細の閲覧のみ**。ログイン不要・全機能無料。

### 5.1 ゲーム検索・一覧

- ゲーム名(日本語・英語)、パブリッシャー名でのキーワード検索
- フィルタ:
  - 配信可否(可 / 条件付き可 / 不可 / 不明)
  - 収益化可否(可 / 条件付き可 / 不可 / 不明)
  - パブリッシャー
  - ジャンル
- 並び替え: 名前順 / 最終確認日順 / 追加日順
- 検索結果は一覧カードで「配信可否」「収益化可否」がひと目でわかるバッジ表示

### 5.2 ゲーム詳細ページ

許諾情報は**原則すべて項目化**し、条件付きの場合の詳細は備考に記載する。

| 項目 | 型 | 内容 |
|------|-----|------|
| 基本情報 | — | タイトル(日/英)、パブリッシャー、発売日、Steam App ID、ジャンル |
| 配信可否 | 選択式 | 可 / 条件付き可 / 不可 / 不明 |
| 収益化可否 | 選択式 | 可 / 条件付き可 / 不可 / 不明 |
| ネタバレ制限 | 選択式 | なし / あり / 不明 |
| 音楽・BGM の扱い | 選択式 | 問題なし / 一部ミュート推奨 / 制限あり / 不明 |
| 切り抜き・アーカイブ | 選択式 | 可 / 条件付き可 / 不可 / 不明 |
| 事前申請 | 選択式 | 不要 / 必要 / 不明 |
| 備考 | テキスト | 条件付きの場合の具体的条件、注意点(例:「発売後 2 週間はエンディング配信禁止」) |
| 根拠 | URL(複数可) | 公式ガイドライン / EULA / 公式 FAQ 等へのリンク(必須)+ 情報種別 |
| 鮮度情報 | 日付 | 運営による最終確認日、ガイドライン改定日(判明している場合) |

### 5.3 パブリッシャー別ページ

- パブリッシャーごとの包括ガイドライン(ある場合)の要約と、対象ゲーム一覧
- 包括ガイドラインを持つパブリッシャーの場合、ゲーム個別設定がなければ既定値を継承(表示上「パブリッシャー包括ガイドラインに準拠」と明示)

### 5.4 静的ページ

- サービスについて / 免責事項・利用規約 / お問い合わせ

## 6. ビジネスモデル

- **Google AdSense によるサイト内広告**
- **パブリッシャーからの広告出稿**(新作タイトルのプロモーション枠等)
- 広告であることの明示(ステマ規制対応)。広告掲載がゲームの可否判定の中立性に影響しないことをポリシーとして明文化する

## 7. 技術スタック(選定)

SEO 最優先・運営 1 名での省力運用・低コストを基準に選定。

| レイヤ | 技術 | 選定理由 |
|--------|------|---------|
| フレームワーク | **Next.js(App Router)+ TypeScript** | 詳細ページの SSG/ISR で SEO 最適化。ゲーム追加時も再ビルド不要(ISR) |
| ホスティング | **Vercel** | Next.js との親和性、無料枠から開始可能 |
| DB・バックエンド | **Supabase(PostgreSQL)** | RDB として許諾情報の構造化に適合。Row Level Security で公開データと管理データを分離 |
| 管理画面 | Next.js 内の管理ルート(Supabase Auth で運営のみアクセス) | 別システムを持たず省力化。Steam API からのメタデータ取込ボタンを実装 |
| ゲームメタデータ | Steam Web API / Storefront API | タイトル・パブリッシャー・発売日・ジャンルの自動取得 |
| スタイリング | Tailwind CSS | 開発速度 |
| 広告 | Google AdSense | §6 参照 |

## 8. データモデル(概要)

```
Publisher(パブリッシャー)
  - name, name_en, official_site_url
  - guideline_url, guideline_summary
  - default_streaming_status, default_monetization_status  ※包括ガイドラインの既定値(ない場合は null)

Game(ゲーム)
  - title, title_en, publisher_id, release_date, genres[]
  - steam_app_id(Steam メタデータ連携キー)
  - streaming_status:    allowed | conditional | prohibited | unknown
  - monetization_status: allowed | conditional | prohibited | unknown
  - spoiler_restriction: none | restricted | unknown
  - music_restriction:   ok | partial_mute | restricted | unknown
  - clip_archive_status: allowed | conditional | prohibited | unknown
  - prior_application:   not_required | required | unknown
  - notes(備考: 条件付きの場合の具体的条件)
  - last_verified_at(運営の最終確認日)

Source(根拠情報、Game に複数紐づく)
  - game_id, url, source_type(guideline | eula | faq | dev_statement | other)
  - noted_at(その情報の日付・改定日)
```

## 9. 非機能要件

- **信頼性が最重要**: すべての可否判定に一次情報リンクと確認日を必須とする。根拠のないデータは登録しない
- レスポンス: 検索・詳細表示は体感即時(ISR + エッジキャッシュ)
- モバイル対応: 配信準備中にスマホで確認するユースケースを想定し、モバイルファースト
- SEO: 「(ゲーム名) 配信 収益化」「(ゲーム名) 配信 ガイドライン」での検索流入が主要導線。詳細ページごとに構造化データ(JSON-LD)・OGP を整備
- 運用: 管理画面から非エンジニア水準の操作でデータ登録・更新が完結すること

## 10. 法的・免責に関する要件(重要)

- 本サービスは **参考情報の提供** であり、法的助言ではないことを全ページで明示する
- 「最終的には必ず公式情報を確認してください」の注意書きと公式リンクを詳細ページに常設
- ガイドライン・EULA 本文の全文転載はしない(著作権配慮)。要約+リンクに留める。翻訳掲載も全文は避け、要点の要約とする
- **ゲーム画像(Steam カプセル画像)の利用方針**(2026-07-14 調査):
  - Steam Web API 利用規約は「Steam Data をサードパーティのサイトでエンドユーザーに表示する」ことを想定して許諾しており、appdetails が返す画像 URL の表示はこの範囲と解釈できる。ただし画像利用を明示的に許諾する条文はない
  - 遵守事項: ①Valve 名・リンクの表示(帰属表示)、②Valve/Steam が本サービスを承認・提携していると誤認させる見せ方の禁止、③API コール上限 10 万回/日、④Steam ロゴ・商標をサイトの主要な要素として使わない(テキストリンク「Steam で見る」が安全)
  - 画像自体の著作権は各パブリッシャーに帰属する。明示ライセンスはないため、**再ホスティングせず Steam CDN の URL を直接参照(ホットリンク)**し、権利者から削除要請があれば即時対応する運用とする(SteamDB 等の主要サードパーティサイトと同等の慣行)
  - 詳細ページから該当ゲームの Steam ストアページへ必ずリンクする(プロモーション的利用であることを明確化)
- 情報の誤り・改定遅れによる損害への免責事項を利用規約に明記
- 広告(AdSense・パブリッシャー出稿)と編集コンテンツの分離を明示

## 11. MVP スコープ外(将来検討)

- ユーザー登録・お気に入り・ガイドライン変更通知
- ガイドライン変更履歴の表示
- ユーザーからの情報提供・誤り報告フォーム(※問い合わせメールで代替)
- 自動クロールによるガイドライン改定検知
- 英語版 UI(海外配信者向け逆展開)
- 事務所向け管理機能、API 提供

## 12. 成功指標(案)

- 掲載タイトル数: リリース時 100 タイトル以上(うち海外 Steam タイトル 70 以上)
- 検索流入からの月間 UU
- 詳細ページから公式情報への遷移率(=信頼して使われている指標)
- AdSense 収益(サーバーコストの回収ライン)
