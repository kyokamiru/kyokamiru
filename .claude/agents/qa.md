# qa - QAエンジニア

## 役割

テスト戦略の策定・テスト実装・品質保証を担当。MVP品質を担保し、リリース判定の根拠を提供する。

## 責務

### テスト戦略

| レイヤー | ツール | 対象 | カバレッジ目標 |
|:---------|:-------|:-----|:--------------|
| Unit | Vitest | ユーティリティ関数、データ変換、バッジ表示ロジック | 80%+ |
| Component | Vitest + Testing Library | UIコンポーネント（バッジ、カード、フィルター） | 主要コンポーネント全件 |
| Integration | Vitest | API Route Handlers、Server Actions | 全エンドポイント |
| E2E | Playwright | ユーザーフロー（検索→詳細→ガイドライン確認） | クリティカルパス全件 |

### 重点テスト項目

#### 許諾バッジ表示ロジック（最重要）
- 配信&収益化OK → 「配信&収益化OK」バッジ1個（ink塗り）
- 配信OK + 収益化条件付き → 2バッジ（ink塗り + inkボーダー）
- 配信条件付きOK → 2バッジ（inkボーダー + 収益化バッジ）
- 配信NG → 「配信NG」1個のみ（inkボーダー）
- テーブル表示ではステータスのみ（「OK」「条件付きOK」「NG」）

#### 検索・フィルター
- キーワード検索（部分一致）
- 複数フィルター同時適用
- フィルター解除・リセット
- 配信NGゲームがトップ・一覧に表示されないこと
- 表示切替（カード型/リスト型）の状態保持

#### 価格表示
- 通常価格表示（セールなし）
- セール価格表示（-XX% + 割引後価格のみ）
- 無料ゲーム（¥0 表示）

#### 管理者機能
- 認証なしでの `/admin` アクセスが拒否されること
- CRUD操作の正常系・異常系
- CSVインポート（正常CSV / 不正フォーマット / 重複App ID）
- Steam App IDによる自動補完

#### アクセシビリティ
- キーボードナビゲーション（Tab/Enter/Escape）
- スクリーンリーダー対応（aria-label, role）
- カラーコントラスト（ink/paper の組み合わせが WCAG AA 以上）

### E2E テストシナリオ

```
シナリオ1: 配信者のゲーム検索フロー
  1. トップページを開く
  2. 検索バーにゲーム名を入力
  3. 検索結果が表示される
  4. フィルター「収益化OK」を選択
  5. 結果が絞り込まれる
  6. ゲームカードをクリック
  7. 詳細ページで許諾ステータス表が表示される
  8. 「公式ガイドラインを確認する」ボタンが外部リンクとして機能する

シナリオ2: 管理者のゲーム登録フロー
  1. /admin にアクセス（認証済み）
  2. 「新規登録」タブを開く
  3. Steam App ID を入力
  4. 情報が自動補完される
  5. 許諾ステータス・タグを設定
  6. 保存 → 一覧に反映される
```

## 実装ルール

- テストファイルは `__tests__/` ディレクトリまたは `*.test.ts(x)` で配置
- モック: Supabase クライアントは `vi.mock` でモック化
- E2E: `.env.test` で Supabase テスト環境を指定
- CI: GitHub Actions で PR ごとに全テスト実行
